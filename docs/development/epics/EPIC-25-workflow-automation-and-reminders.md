# EPIC-25: Workflow Automation And Reminders

## Goal

Turn the new local queue into a practical recurring workflow surface without external automation or hidden browser actions.

## Inputs

- `docs/Техническое заданиеV.1.md` sections `Phase 3`, `24.3`
- current repository state after `EPIC-22` readiness

## In Scope

- kanban-style queue view;
- manual stage actions and queue ergonomics;
- local reminders and daily summary surfaces;
- workflow event summaries built only from local data;
- focused tests for queue transitions and reminder logic.

## Explicitly Deferred

- background queue processing;
- webhook delivery;
- HR communication hub;
- multi-site expansion;
- public release packaging.

## Success Criteria

- the queue becomes a repeatable local workflow rather than only a list view;
- reminder and summary logic helps the user act without automating HH;
- the product stays local-first and read-first.

# EPIC-32: Private Release Readiness

## Goal

Make VacancyPilot materially easier and safer to install, understand, and verify in private distribution before any public store work begins.

## Inputs

- `docs/Техническое заданиеV.1.md`
- `docs/development/private-install-guide.md`
- `docs/development/public-release-prerequisites.md`
- `docs/development/privacy-policy-checklist.md`
- `docs/development/release-checklist.md`

## In Scope

- onboarding completion for real private installs;
- permission disclosure and privacy explainer surfaces;
- package/install asset hardening needed for disciplined private sharing;
- updated QA/release evidence for the private-install path.

## Explicitly Deferred

- Chrome Web Store submission;
- Firefox/Edge store packaging;
- legal/public policy finalization for a public launch;
- monetization;
- multi-site expansion.

## Success Criteria

- a new private tester can understand what the extension does, does not do, and which permissions it uses;
- private install docs and release checks align with the actual product surface;
- the pack produces a cleaner start point for any later public-release epic.

# EPIC-10: Release Hardening And QA

## Goal

Make Phase 1 safe to use privately and ready for public release preparation.

## Scope

- Manifest audit.
- Privacy/safety tests.
- Fixture regression tests.
- Manual QA checklist.
- Browser compatibility smoke tests.
- Release candidate notes.
- Store readiness checklist.

## Non-Goals

- No public launch until privacy policy and store assets exist.
- No Labs release.
- No monetization work.

## Acceptance Criteria

- CI release gate passes.
- Manifest has only expected permissions.
- Privacy/safety tests pass.
- Manual QA results are documented.
- Known risks are listed before release.

## Safety Notes

Release hardening is not polish. It is the point where account safety, data safety, and extension review risk are verified.

# EPIC-34: Workflow UX Refinement

## Goal

Improve the day-to-day usability of the core workflow surfaces so repeated vacancy triage, tracking, and communication work is faster, clearer, and less error-prone.

## Inputs

- `docs/Техническое заданиеV.1.md`
- current repository state after `EPIC-33`
- existing workflow components such as kanban, HR workspace, guided apply, popup, side panel, and search triage

## In Scope

- ergonomics of frequent actions, status changes, filters, and contextual information;
- clearer interaction feedback, warnings, disabled states, and action grouping;
- accessibility-oriented polish for keyboard flow, labels, and visible state changes;
- focused responsive improvements where workflow surfaces are cramped.

## Explicitly Deferred

- new backend automation;
- new HH automation capabilities;
- broad data-model changes unrelated to UX;
- public-release/store packaging work.

## Success Criteria

- repeated daily actions require less hunting and less context switching;
- runtime surfaces provide clearer feedback when actions succeed, fail, or are unavailable;
- workflow views become easier to scan under real extension-size constraints;
- no forbidden HH automation boundaries are weakened.

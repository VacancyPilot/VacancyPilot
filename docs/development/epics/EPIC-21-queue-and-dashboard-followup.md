## Goal

Extend the product after the search-surface MVP with a local queue view, stronger dashboard workflows, company greylist support, and duplicate detection.

## Inputs

- `docs/Техническое заданиеV.1.md` sections `Phase 2`, `24.2`
- current repository state after `EPIC-20`

## In Scope

- local queue as a task list;
- stronger dashboard workflows for search-triage outputs;
- company greylist controls;
- duplicate detection heuristics using local data;
- focused tests for new queue/dashboard logic.

## Explicitly Deferred

- auto-apply;
- hidden tabs or background vacancy opening;
- HR communication hub;
- guided apply labs;
- multi-site expansion.

## Success Criteria

- saved/search-triaged vacancies can be organized as work items locally;
- the dashboard becomes useful for repeated triage and review;
- Phase 2 remains read-first and local-first.
